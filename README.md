# Data Processing & Aggregation Pipeline

Локальная платформа для генерации, обработки и распределенной агрегации данных. Развернута в изолированной контейнерной среде. Основной упор сделан на применение механизмов отказоустойчивости (Fault Tolerance) на уровне планировщика задач и вычислительного кластера.

## 🛠 Стек технологий
* **Оркестрация:** Apache Airflow (LocalExecutor)
* **Распределенные вычисления:** Apache Spark 3.5.1 (Standalone Cluster: 1 Master, 2 Workers)
* **Хранилище:** ClickHouse (колоночная СУБД для метрик) + PostgreSQL 15 (метаданные Airflow)
* **Инфраструктура:** Docker Compose

## Архитектура конвейера

Логика пайплайна описана в DAG `ml_feature_pipeline` и состоит из трех последовательных шагов:

1. **`generate_data_task`**: Генерация синтетического датасета (10 000 строк логов сессий: `user_id`, длительность, клики). Данные сохраняются в CSV, путь к файлу пробрасывается в следующую таску через механизм XCom.
2. **`process_data_task`**: Симуляция обогащения данных через нестабильное внешнее API. Для проверки механик оркестратора в код заложена генерация исключения с вероятностью 70%. Airflow настроен на 10 повторных попыток (`retries`) с задержкой 10 секунд.
3. **`spark_aggregation_task`**: Запуск PySpark-джобы через `SparkSubmitOperator`. Выполняется группировка по `user_id` и расчет метрик. Результат отправляется в ClickHouse прямым HTTP POST-запросом в формате `CSVWithNames`.

## Быстрый старт

**1. Клонирование и запуск кластера**
```bash
git clone <URL_репозитория>
cd <название_папки>
docker compose up -d
```

**2. Доступ к интерфейсам**
* **Spark Master UI:** `http://localhost:8080` (статус воркеров и запущенных джоб)
* **ClickHouse Web UI:** `http://localhost:8123/play` (встроенная консоль для SQL-запросов, логин `default`)
* **Airflow UI:** `http://localhost:<порт_airflow>` (запуск DAG)



### 3. Проверка результатов
После успешного завершения пайплайна проверьте витрину в ClickHouse через браузер (`http://localhost:8123/play`):
```sql
SELECT * FROM default.user_aggregated_features LIMIT 10;
```
